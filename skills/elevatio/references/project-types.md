# Project Types - Classification Guide

## Type Classification Criteria

### 1. Learning Type

**Keyword Patterns:**
- learn, study, course, tutorial, guide, beginner
- Coursera, Udemy, YouTube learning
- master, understand, practice

**Characteristics:**
- Goal is acquiring new skills/knowledge
- Curriculum/roadmap needed
- Progress tracking required

**Example:** "Want to learn Arduino"

---

### 2. Development Type

**Keyword Patterns:**
- build, develop, create, implement, tool, automation
- app, plugin, script, API
- make, code, program

**Characteristics:**
- Goal is working deliverable
- Tech stack/architecture needed
- Version control/deployment considered

**Example:** "Build automation tool"

---

### 3. Research Type

**Keyword Patterns:**
- research, analyze, investigate, compare, explore
- paper, reference, data
- study, examine, survey

**Characteristics:**
- Goal is knowledge/insight derivation
- Data collection/organization needed
- Conclusion/report output

**Example:** "AI trend analysis"

---

### 4. Creative Type

**Keyword Patterns:**
- create, design, art, video, music, writing
- content, portfolio, artwork
- visual, audio, media

**Characteristics:**
- Goal is creative output
- Inspiration/reference collection needed
- Iterative revision/improvement

**Example:** "TouchDesigner interactive art"

---

## Handling Mixed Types

When note matches multiple types:

1. **Select 1 primary type** (user confirmation)
2. **Add secondary type tag**
3. **Reference all applicable guides**

Example: "Making music with AI"
- Primary: Creative (music production)
- Secondary: Learning (AI tool learning)
