# Archive Checklist

> 프로젝트 아카이브 전 확인사항

---

## Pre-Archive Verification

### 1. 완료 상태 확인

```
□ README.md Progress Tracker 확인
□ 미완료 항목 존재 시 사용자 확인
□ 강제 아카이브 가능 (사용자 승인 필요)
```

### 2. 파일 정리 상태

```
□ 임시 파일 정리됨
□ 불필요한 스크립트 제거됨
□ logs/ 폴더 최신 상태
```

### 3. 링크 확인

```
□ 다른 프로젝트에서 이 프로젝트 참조하는지 확인
□ 참조 있으면 아카이브 후에도 Wikilink 유지됨 (Obsidian 자동 처리)
```

---

## Archive Reasons

| Reason | 설명 | Graveyard 기록 |
|--------|-----|---------------|
| `completed` | 목표 달성 | ❌ 불필요 |
| `discontinued` | 중단 | ✅ 필수 |
| `on hold` | 일시 중지 | ✅ 필수 |
| `merged` | 병합됨 (via unio) | ❌ 불필요 |

---

## Project-Specific Checklist

```
□ Completed: 모든 Phase/Step 완료, 최종 결과물 존재
□ Completed: 회고/정리 노트 작성 (선택)
□ Discontinued/On Hold: 중단 사유 + 교훈 기록 (Graveyard)
□ Discontinued/On Hold: 재개 가능성 있으면 on hold 선택
□ All: Connected Projects 링크 정리
```

---

## Post-Archive Verification

아카이브 완료 후:

```
□ 폴더가 54. Archive/로 이동됨
□ Registry에 archived 상태 반영됨
□ Graveyard 기록 완료 (해당 시)
□ Daily Note에 보고됨
```
